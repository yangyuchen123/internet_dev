# AI Agent Platform

简体中文说明文档 — 一个用于快速搭建和开发 AI Agent 平台的工程模板。

## 📁 目录结构

```
├── backend/     # 后端服务（API、Agent 管理、模型接入等）
├── frontend/    # Web 前端（管理控制台、监控、调试界面）
└── mini-app/    # 小程序或轻量客户端示例
```

## 🚀 项目简介

这是一个用于构建 AI Agent 平台的参考实现/模板，包含后端、前端和小程序示例。

## ✨ 主要功能

- **Agent 管理** - 完整的生命周期管理（创建、启动、停止、监控）
- **多模型支持** - 本地模型 / 云模型适配层
- **插件体系** - 可扩展的插件与策略体系
- **可视化控制** - 前端控制台用于可视化管理与调试

## 📋 自动化脚本使用指南

### 前提条件
- ✅ 已正确配置 `.env` 文件（基于 `.env_temp` 模板创建）
- ✅ 本地环境已安装 Git 并配置好用户信息

### 🛠️ 可用脚本说明

| 脚本名称 | 功能 | 使用方法 |
|---------|------|----------|
| `push_to_my_remote.bat` | 提交并推送到个人远程仓库 | 双击运行 → 输入提交信息 → 自动执行 |
| `sync_local_with_my_remote.bat` | 同步本地与个人远程仓库 | 双击运行 → 自动拉取最新代码 |
| `sync_my_remote_with_upstream.bat` | 同步上游仓库到个人仓库 | 双击运行 → 自动合并并推送 |
| `create_pr_to_upstream.bat` | 创建 PR 到原主仓库 | 双击运行 → 浏览器打开 PR 页面 |

### 🔄 建议工作流程

1. **开始工作前**
   ```bash
   sync_my_remote_with_upstream.bat  # 同步上游最新代码
   ```

2. **完成工作后**
   ```bash
   push_to_my_remote.bat  # 提交到个人仓库
   ```

3. **准备贡献**
   ```bash
   create_pr_to_upstream.bat  # 创建 PR
   ```

4. **定期维护**
   ```bash
   sync_local_with_my_remote.bat  # 保持本地与远程同步
   ```

### ⚠️ 注意事项

- 🚨 首次使用前必须正确配置 `.env` 文件
- ⚠️ 合并冲突时需要手动解决
- 🔒 运行脚本前确保没有未提交的重要修改
- 🌐 网络问题请检查 Git 配置和连接
